Проведи code review текущей ветки относительно ветки `main` согласно правилам проекта.

## Шаги

1. Определи имя текущей ветки и номер PR из неё (например, `feature/RC-2` → PR `rc-2`).

2. Определи текущую дату и время в формате `YYYY-MM-DD_HH-MM` (используй `date +%Y-%m-%d_%H-%M`).

3. Получи данные ревьюера из git config:
   ```
   git config user.name
   git config user.email
   ```

4. Получи список изменённых файлов и диффов относительно `main`:
   ```
   git diff main...HEAD --name-only
   git diff main...HEAD
   ```

5. Прочитай все правила из `.claude/rules/` (php-style.md, symfony.md, clean-architecture.md, cqrs.md).

6. Прочитай все изменённые исходные файлы (не тесты, не миграции) и проверь каждый из них по всем правилам.

7. Составь отчёт в формате Markdown со следующей структурой:

```markdown
# Code Review — [branch-name]
**Дата:** [дата и время]
**Ветка:** [branch] → main
**Файлов изменено:** N
**Ревьюер:** [user.name] <[user.email]>

## Критические нарушения
[Список нарушений с указанием файла, строки и ID правила. Если нет — написать «Нет».]

## Предупреждения
[Менее критичные отступления. Если нет — написать «Нет».]

## Замечания
[Стилистика, именование, незначительные улучшения. Если нет — написать «Нет».]

## Итог
[1–3 предложения: общая оценка качества кода по правилам проекта.]
```

8. Сохрани отчёт в файл:
   ```
   docs/reports/pull-requests/[pr-name]/[YYYY-MM-DD_HH-MM].md
   ```
   Директорию создай, если её нет.

9. Сообщи пользователю путь к сохранённому отчёту и выведи краткое резюме (только секции «Критические нарушения» и «Итог»).
