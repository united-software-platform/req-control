## Язык и терминология

**[DOC-001]** Весь текст документации, комментарии в коде и текстовые ответы пишутся на **русском языке**.

**[DOC-002]** Идентификаторы кода — имена переменных, классов, методов, файлов — остаются на английском.

**[DOC-003]** Понятия без устоявшегося русского перевода не переводятся: Use Case, Value Object, Aggregate Root; названия инструментов: Doctrine, PHPUnit, Symfony.

---

## Структура файла

**[DOC-014]** Каждый файл открывается тремя обязательными элементами: заголовок, назначение (1–3 строки), секция навигации.

```markdown
# Название документа

Краткое назначение (1–3 строки).

---

## Навигация

- [Раздел 1](#раздел-1)
- [Раздел 2](#раздел-2)

---
```

**[DOC-012]** Уровни заголовков не пропускаются: строго `#` → `##` → `###` → `####`.

**[DOC-009]** Текст разбит на логичные короткие разделы — без "простыней" сплошного текста.

**[DOC-010]** Один раздел — один вопрос. Если раздел охватывает два вопроса, разделить его на два.

**[DOC-011]** Объёмные темы выносятся в отдельный файл; в основном документе — краткое резюме и ссылка "Подробнее".

**[DOC-013]** Маркированные списки — дефис `-`, нумерованные — `1.`. Вложенность не глубже двух уровней.

---

## Разметка Markdown

**[DOC-004]** Формат — CommonMark / GitHub-style Markdown, совместимый с GitLab и Obsidian. Нестандартные расширения не используются.

**Запрещено:**

- **[DOC-005]** HTML-теги: `<div>`, `<span>`, HTML-таблицы, `<h1>..<h6>`, `<a name="...">`
- **[DOC-006]** Wikilinks: `[[Note]]`
- **[DOC-007]** Четыре пробела в качестве отступа вместо fenced code blocks
- **[DOC-008]** Подчёркивание заголовков: `====` / `----`

## Элементы оформления

### Ссылки [DOC-015]

**[DOC-015]** Все ссылки — только относительные: `[текст](./path/file.md#anchor)`.

Запрещены: абсолютные URL на репозиторий, Wikilinks `[[...]]`, HTML-якоря `<a href="...">`.

### Код [DOC-016]

**[DOC-016]** Код оформляется исключительно в fenced code blocks с обязательным указанием языка.

````markdown
```php
echo "Привет";
```

```bash
composer install
```
````

### Таблицы [DOC-017]

**[DOC-017]** Таблицы — стандартный Markdown `|...|`. HTML-таблицы запрещены.

### Диаграммы [DOC-018]

**[DOC-018]** Основной формат диаграмм — Mermaid. Для сложных архитектурных C4 — PlantUML в отдельных `.puml` файлах.

Требования к читаемости:
- Не более 7–10 узлов на одной диаграмме; при превышении — разбивать
- Связи подписаны: что передаётся, какой протокол
- Логические группы оформляются через `subgraph` (Mermaid) или `Boundary` (PlantUML)

### Callout-блоки [DOC-019]

**[DOC-019]** Выделение важной информации — через blockquote с текстовым маркером.

```markdown
> **Важно:** Критическая информация, которую нельзя пропустить.
> **Внимание:** Потенциальная проблема или ограничение.
> **Примечание:** Дополнительная информация для контекста.
```

## Полнота содержания

**[DOC-020]** Ограничения, допущения, риски и неподдерживаемые сценарии фиксируются явно, а не подразумеваются.

## Жизненный цикл документа

**[DOC-022]** Каждый md-файл завершается таблицей версий:

```markdown
## Версионирование

| Версия | Дата | Задача | Описание изменений |
|--------|------|--------|--------------------|
| 1.0.0 | 2026-02-19 | Задача #4 | Начальное создание |
| 1.0.1 | 2026-02-25 | Задача #7 | Добавлен раздел X |
```

Нумерация: `1.0.0` — первая версия; `1.0.1`, `1.0.2` — частичные изменения; `1.2.0` — кардинальная переработка.

**[DOC-023]** При переносе контента в другой файл — исходный материал допустимо сжать, но ключевые факты и логика сохраняются. В файле-источнике остаётся ссылка на новое расположение.
