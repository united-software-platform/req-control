# Правила шаблонов

Стандарты оформления файлов-шаблонов. Распространяются на все документы, идентифицированные как шаблоны.

## Таблица правил

| ID | Правило |
|----|---------|
| TMPL-001 | Шаблон определяется по постфиксу `template` в имени файла или по frontmatter `type: template` |
| TMPL-002 | Шаблоны не содержат артефактов конкретных проектов |
| TMPL-003 | Все переменные значения заменяются плейсхолдерами `{{...}}` |

---

## Распознавание шаблона

**[TMPL-001]** Файл является шаблоном при выполнении хотя бы одного из условий:

- имя файла оканчивается на `template` (например, `system-specification-template.md`)
- файл содержит frontmatter с полем `type: template`:

```markdown
---
type: template
description: "{описание шаблона}"
---
```

---

## Требования к содержанию

**[TMPL-002]** Шаблон не несёт данных конкретного проекта.

Запрещено включать:

- названия проектов, систем, организаций
- имена и email-адреса сотрудников
- реальные URL, IP-адреса, хостнеймы
- конкретные номера задач, тикетов, версий
- любые данные, специфичные для окружения или команды

**[TMPL-003]** Каждое переменное значение заменяется плейсхолдером в двойных фигурных скобках `{{...}}`.

```markdown
---
type: template
description: "Шаблон технического задания"
---

# {{НАЗВАНИЕ_ПРОЕКТА}}

Автор: {{АВТОР}}
Дата: {{ДАТА}}
Задача: {{НОМЕР_ЗАДАЧИ}}
```
